## Projeto Oferta de Disciplina (Back-End)
> Desenvolvendo a API Application do Pojeto

### Informações Importantes

* A atualização do ponteiro main é controlada
> A cada encerramento de versão, "main" será atualizado sempre a partir do "develop"

* os resultados validados dos "features" devem atualizar "develop"
> O ponteiro "develop" é o resultado das "features"
>> feature devem conter as entregas mais específicas do projeto, que ao concluir (e devidamente validado) deve atualizar "develop"

* Uso do git flow recomendado
> 01. git clone git@github.com:ucsal/ofertadisciplina_back.git
> 02. acessar o repositorio criado
> 03. git flow init (mantenham os mesmos nomes, por padrão para facilitar o controle)
> 04. branch production - main
> 05. branch development - develop
> 06. branches support - feature, bugfix, release, hotfix, support
> 07. tag previx [] (deixa em branco mesmo)
> 08. faça o push sempre nos repositorios das "features"
> 09. ao finalizar todo trabalho do repositorio, precisamos validar o que foi feito
> 10. uma vez validado finalizamos o repositório e atualizamos o develop
